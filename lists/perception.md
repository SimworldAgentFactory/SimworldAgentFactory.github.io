The perception module converts heterogeneous environment observations into a standardized representation for downstream modules. Raw observations may include symbolic state, sensor readings, RGB frames, and textual feedback. The perception module normalizes these inputs into a unified state representation, typically expressed as structured JSON-like fields together with concise textual summaries that are easy for language-model-based modules to consume.This design provides a clean interface between environment dynamics and downstream decision-making. By separating state abstraction from downstream control, perception allows the same memory, reasoning, and reflection components to operate across environments with very different observation formats while preserving task-relevant structure and abstracting away environment-specific details.
